

Build a production-ready AI-powered QA Copilot web application.

## Context

I am a Senior QA Engineer with 5.5 years of experience.

The application should help QA engineers automate requirement analysis and test documentation creation.

The primary workflow is:

1. Read Jira ticket
2. Analyze PRD / Acceptance Criteria
3. Generate Test Plan
4. Download Test Plan as DOCX
5. Generate Test Cases from Test Plan
6. Download Test Cases as CSV

The application should be modern, lightweight, fast, responsive, and visually impressive.

Use app.vwo.com as UI inspiration.

---

# Core Workflow

User enters Jira ID.

Example:

SCRUM-7

System should:

1. Connect to Jira API

2. Fetch Jira Story

3. Read:

   * Summary
   * Description
   * Acceptance Criteria
   * Comments
   * Labels
   * Priority
   * Attachments Metadata

4. AI analyzes the requirement

5. Generate Test Plan

6. Allow DOCX download

7. Generate Test Cases

8. Allow CSV download

# CSV Format

Use exactly these columns:

Scenario

TID

Test Data

TestCase Description

PreCondition

TestSteps

Expected Result

Actual Result

Steps to Execute

Expected Result

Actual Result

Status

Executed QA Name

Example Row:

Scenario:
Login

TID:
TC_LOGIN_001

Test Data:
valid_user

TestCase Description:
Verify successful login

PreCondition:
User account exists

TestSteps:
Enter valid username and password

Expected Result:
Login successful

Actual Result:
Blank

Steps to Execute:

1. Open application
2. Enter username
3. Enter password
4. Click Login

Expected Result:
Dashboard displayed

Actual Result:
Blank

Status:
Not Executed

Executed QA Name:
Blank

Output filename:

SCRUM-7_TestCases.csv
